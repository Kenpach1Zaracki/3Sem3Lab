**Лабораторная №3 — Структуры данных (C++ / Go)**

- **Описание:** Учебная реализация базовых структур данных с тестами и бенчмарками.

**Цель проекта**: показать работу и протестировать реализации основных структур данных: двоичное дерево поиска (BST), одно- и двусвязные списки, стек, очередь, хеш-таблицы/множества. Код предназначен для обучения и содержит модульные тесты и бенчмарки.

**Реализовано**
- **BST:** двоичное дерево поиска (C++).
- **Списки:** одно- и двусвязные списки (C++ и Go).
- **Стек:** реализация стека (C++ и Go).
- **Очередь:** реализация очереди (C++ и Go).
- **Хеш-таблица / Множество:** базовая реализация хеш-структур (C++).

**Структура репозитория**
```
.
├── src/              # Реализация на C++ (заголовки и исходники)
├── tests/            # Тесты на C++ (Google Test)
├── bench/            # Бенчмарки на C++ (Google Benchmark)
├── golang/           # Реализации на Go и их тесты
│   └── containers/   # Go-пакет с контейнерами
├── docs/             # Документация и UML диаграммы
├── build/            # Директория сборки (создается автоматически)
│   ├── bin/          # Скомпилированные бинарники
│   ├── obj/          # Объектные файлы
│   └── coverage/     # HTML отчеты покрытия кода
├── Makefile          # Система сборки C++ проекта
├── run_go_tests.sh   # Скрипт для запуска Go тестов
└── .gitignore        # Игнорируемые файлы
```

**Makefile**

- `all` (по умолчанию) — собрать основную программу `build/bin/app`.
- `clean` — удалить директорию `build/` и все временные артефакты.
- `tests` — собрать тестовый бинарник `build/bin/tests_bin` с флагами покрытия.
- `run_tests` — собрать и запустить тесты.
- `coverage` — запустить тесты и сгенерировать отчёт покрытия в `build/coverage/coverage.html`.
- `bench` — собрать бенчмарки в `build/bin/bench_bin`.
- `run_bench` — собрать и запустить бенчмарки.

**Примеры команд**
```bash
# Собрать основную программу
make
# Запустить: ./build/bin/app

# Собрать и запустить тесты
make run_tests

# Сгенерировать отчет покрытия кода
make coverage
# Открыть: build/coverage/coverage.html

# Собрать и запустить бенчмарки
make run_bench

# Очистить результаты сборки
make clean

# Тесты Go
./run_go_tests.sh
# или
cd golang && go test ./...
```

**Требования**
- C++17 компилятор (g++/clang++)
- Google Test для тестов
- Google Benchmark для бенчмарков
- gcovr для генерации отчетов покрытия
- Go 1.18+ для Go-реализаций

**Примечания**
- Все артефакты сборки помещаются в директорию `build/` и не коммитятся в git
- UML диаграммы и документация находятся в `docs/`
- Пути к Google Test и Google Benchmark настраиваются в `Makefile` (переменные `GTEST_ROOT` и `BENCH_ROOT`)
