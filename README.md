**Лабораторная №3 — Структуры данных (C++ / Go)**

- **Описание:** Учебная реализация базовых структур данных с тестами и бенчмарками.

**Цель проекта**: показать работу и протестировать реализации основных структур данных: двоичное дерево поиска (BST), одно- и двусвязные списки, стек, очередь, хеш-таблицы/множества. Код предназначен для обучения и содержит модульные тесты и бенчмарки.

**Реализовано**
- **BST:** двоичное дерево поиска (C++).
- **Списки:** одно- и двусвязные списки (C++ и Go).
- **Стек:** реализация стека (C++ и Go).
- **Очередь:** реализация очереди (C++ и Go).
- **Хеш-таблица / Множество:** базовая реализация хеш-структур (C++).

**Структура репозитория**
- `src/` : реализация на C++ (заголовки и исходники).
- `tests/` : тесты на C++.
- `bench/` : бенчмарки на C++.
- `golang/` : реализации на Go и их тесты (`golang/containers/`).
- `Makefile`, `run_go_tests.sh` : вспомогательные скрипты для сборки и запуска тестов.

Запуск тестов Go:
```zsh
./run_go_tests.sh
# или вручную
cd golang
go test ./...
```

Запуск бенчмарков (пример ручной сборки):
```zsh
g++ -O2 bench/bench_main.cpp -o bench/bench_main
./bench/bench_main
```

**Makefile**

- `all` (по умолчанию) — собирать основную программу `app`.
- `clean` — удалить объектные файлы, бинарники и временные артефакты.
- `tests` — собрать тестовый бинарник `tests_bin` с флагами покрытия (`--coverage`).
- `run_tests` — собрать тесты и запустить `./tests_bin`.
- `coverage` — запустить `run_tests`, затем сгенерировать отчёт покрытия `coverage.html` (используется `gcovr`).
- `bench` — собрать бенчмарки в бинарник `bench_bin`.
- `run_bench` — собрать бенчмарки и запустить `./bench_bin`.

Примеры команд (macOS / zsh):
```zsh
# собрать основную программу
make

# собрать и запустить тесты
make tests
make run_tests

# собрать и получить отчет покрытия
make coverage

# собрать и запустить бенчмарки
make bench
make run_bench

# очистить результаты сборки
make clean
```

Примечания:
- Тесты собираются с использованием `googletest` (в Makefile заданы пути `GTEST_ROOT`).
- Бенчмарки собираются с использованием `google-benchmark` (путь в `BENCH_ROOT`).
- Для корректной генерации покрытия требуется `gcovr`.
